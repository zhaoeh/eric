mule常用知识点总结 1、mule操作db的批操作 
        原生jdbc的Statement 和 PreparedStatemen 对象都提供批量操作，通过指定executeBatch来一次性缓存多条insert、update、delete语句等，最后一次性提交给db，避免了和数据库多次建立连接，提升性能。

        jdbc批量操作案例：
        
            // Create SQL statement
            String SQL = "INSERT INTO Employees (id, first, last, age) " + "VALUES(?, ?, ?, ?)";
            ​
            // Create PrepareStatement object
            PreparedStatemen pstmt = conn.prepareStatement(SQL);
            ​
            //Set auto-commit to false
            conn.setAutoCommit(false);
            ​
            // Set the variables
            pstmt.setInt( 1, 400 );
            pstmt.setString( 2, "Pappu" );
            pstmt.setString( 3, "Singh" );
            pstmt.setInt( 4, 33 );
            // Add it to the batch
            pstmt.addBatch();
            ​
            // Set the variables
            pstmt.setInt( 1, 401 );
            pstmt.setString( 2, "Pawan" );
            pstmt.setString( 3, "Singh" );
            pstmt.setInt( 4, 31 );
            // Add it to the batch
            pstmt.addBatch();
            ​
            //add more batches
            ​
            //Create an int[] to hold returned values
            int[] count = stmt.executeBatch();
            ​
            //Explicitly commit statements to apply changes
            conn.commit();
        
        executeBatch的原理很简单，通过一次和数据库的连接，然后每构建好一个SQL语句后，便执行一次pstmt.addBatch()，将这个SQL先缓存到statement对象中。
        等到最终所有目标SQL都通过pstmt.addBatch()缓存到statement对象之后，再统一执行executeBatch()，将所有的SQL一次性执行，达到了批处理的目的。

        批处理的特性，往往会被我们应用到完全相同的SQL语句（只是参数不同）中，然后通过循环来迭代提交，如下案例：
        
            String SQL = "INSERT INTO Employees (id, first, last, age) " + "VALUES(?, ?, ?, ?)";
            while(resultSet.next()){
                pstmt.setInt( 1, resultSet.getInt(1) );
                pstmt.setString( 2, resultSet.getString(2) );
                pstmt.setString( 3, "resultSet.getString(3) );
                pstmt.setInt( 4, resultSet.getInt(4) );
                pstmt.addBatch();
            }
        
        上述案例在模拟一个数据复制的过程，resultSet是从源表中查询出来的数据结果集，然后遍历这个结果集，迭代设置不同的insert参数，每次构建好SQL语句后，进行addBatch操作。
        最终再统一进行executeBatch。

        知道了原理，接下来看mule里面的批处理：
            1.  mule中的select实际上和jdbc、mybatis一样，如果查询不到数据，会自动将结果集映射为一个空的llist集合对象，而不会产生null值。实际上这一点jdbc返回的ResultSet结果集就说明了，ResultSet不会是null值。
            2.  对于insert、update、delete等语句来说，有个批量操作，底层原始使用的就是jdbc的executeBatch：
            通过在insert等标签中添加属性 bulkMode="true"来对当前的语句开启批量处理功能。
            加上bulkMode="true"属性就表示批量操作，此时的入参payload，要求payload是一个list集合对象，如果此时的payload是一个null值的话，那么将会报错。
            如果此时的payload是一个空集合对象（不是null值），此时的批处理将不会进去，也不会报错。
            mule的底层对批处理实际上是通过遍历入参payload来循环设置参数，循环提交SQL语句的，如果payload是空对象，那么循环将进不去。
            3.  重点来了。同样，如果不使用批量操作，即去掉bulkMode="true"，直接进行单个的insert等操作，此时如果传入的payload是个空集合对象，那么对应的参数取出来就会是null。
            这个时候是会执行insert语句的，如果数据库对字段有非空约束，则会插入报错。
            这一点需要注意。
