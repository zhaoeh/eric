Oracle服务器、Oracle客户端、PLSQL Developer 1、区分三个软件系统
        关键字：138.138.2.240:1521/esbdb
        区分Oracle数据库中的三个软件系统:
        Oracle服务器：Oracle 11g，真正需要安装的Oracle Server。
        Oracle客户端：Oracle的客户端。
        PL/SQL Developer第三方工具：用于连接Oracle Server，并进行各种SQL操作的第三方工具。

        1.  说明
        不论是本机Windows系统下安装的Oracle服务器还是远程或者虚拟机的Linux系统下安装的Oracle服务器，只要本机想要访问Oracle服务器，就必须安装Oracle客户端。

        3.  一路“下一步”进行安装，选择本地安装目录
        
        4.  安装好后如下图所示
        
        5.  修改客户端网络服务配置文件
        目录为：D:\soft\application\OracleClient\instantclient_11_2\NETWORK\ADMIN
        进入NETWORK/ADMIN目录下，找到文件tnsnames.ora打开并修改：
        
            该文件是本地网络服务配置文件，一般用来配置客户端的，配置后此客户端就可以访问oracle服务器端。
            总结：oracle服务器端需要配置listen这个监听文件，客户端需要配置tnsnames这个本地网络文件。
            当然，不论是客户端还是服务器端都会存在这两个文件，就看客户端或者服务器端是要充当什么角色了，有时候客户端也能充当服务器端就需要配置listen，有时候服务器端也想充当客户端去访问其他的oracle服务器，那么也需要配置tnsnames这个文件
        
        

        修改后的内容如下：
        
            zhao = //zhao是为下列数据库实例esbkf1起的一个别名，用PL/SQL
             (DESCRIPTION = //登录时访问
             (ADDRESS = (PROTOCOL = TCP)(HOST = 10.17.1.92)(PORT = 1521)) //数据库服务器的Ip
             (CONNECT_DATA =
             (SERVER = DEDICATED)
             (SERVICE_NAME = esbkf1) //esbkf1是Oracle Server端创建的数据库实例名
             )
             )
        
        上述文件节点可以加多个数据库实例映射名，所以可以配置多个节点映射。

        6.  修改完成后，保存该文件并关闭
        7.  可能出现ORA-121541：TNS：无法解析指定的连接标识符 这个错误
        当本文档的所有工作都做完后，使用PLSQL工具进行Oracle数据库的连接，出现上述错误.
        解决办法，将本地客户端的网络配置文件，即上一步的tnsnames.ora文件中，所有数据库监听实例之间的首行都不要存在空格，多个监听节点之间不能存在空格，不能有空行存在，结构必须规范整齐即可解决上述问题。

8、 新建SQL可视化控制台窗口，编写SQL语句并执行
        1.  说明
        *.dmp文件对于Oracle来说就是备份文件，Oracle中所有的表、视图、索引、过程、函数、触发器、包等块程序都能被导出为*.dmp文件，*.dmp文件主要就是用来备份数据库中的数据的，必要时倒入*.dmp文件可以用来恢复数据库；Oracle在sqlplus命令行控制台支持exp命令去导出为*.dmp，支持imp命令去导入*.dmp文件。
        

        2.  可执行文件
        选择imp.exe命令文件：D:\soft\oracle11g\app\zhaoeh\product\11.2.0\dbhome_1\bin\imp.exe
        导入文件选择需要导入的dmp文件：C:\Users\zhaoeh\Desktop\esb2_trans_log_his.dmp
        
        输入后点击右下方的导入按钮即可，等待dos窗口响应，响应自动退出
        

        3.  查看
        导入dmp文件的本质就是向库里面导入了一张表,在哪个用户下导入的，该dmp文件就会导入了哪个用户下的表中。
